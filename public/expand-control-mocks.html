Multi-layer expand / collapse — three mocks

Three patterns for navigating Category → Feature → Entity → Field. The shared table mock below each control bar shows the resulting state.

Option A · Depth stepper
A single 4-step control sets the visible depth globally.
Click a step → the table opens to that level everywhere. Per-row carets still work for selectively collapsing one branch.

Option B · Per-level expand chips
A button per layer plus a single Collapse-all.
"Expand to entities" opens features+entities globally. Carets still let you collapse a single row. More flexible; busier control bar.

Option C · Per-row "expand subtree"
A small ⇲ on each row opens everything beneath it.
Hover a category → click ⇲ → its features, entities, and fields all open. Same icon at the feature level scopes to that one feature. Top-bar Expand-all still exists for global.

Recommendation
Ship Option A (depth stepper) first — cheapest to build, communicates the four-layer model visually, and matches how most data-explorer UIs handle depth. Add Option C's ⇲ later for power users who need to dig into one branch. Option B only if you find users routinely want to see, say, all features but no entities.
