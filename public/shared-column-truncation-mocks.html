Shared-with column — five ways to truncate

All five sit inside a dedicated Shared with column. The difference is how each one handles a long list (e.g. an entity used by 7 features). Pick whichever balance of compactness vs. clickiness you like best.

A · first 3 + "+N more" link (expands inline)
Show first 3 IDs, then a clickable "+N more" that expands the row in place.
Most common web pattern. Default state is short and predictable; clicking gives full detail without leaving the row.

B · count badge + preview, "show all" link
Always lead with a count. Show first 1–2 IDs as preview. "show all" link reveals the rest.
Count is the strongest information ("oh, this is shared 7 times"). The IDs are secondary.

C · ID chips with overflow chip
Each ID is a small pill. Beyond N, a dashed "+4" chip absorbs the rest.
Same compactness as A, but each ID is visually a peer (and clickable on its own to filter/jump).

Recommendation
Go with Option A. It's the most-used pattern on the web, communicates "this is a list with more behind it" instantly, and uses no popover machinery.

If you want the count to be the headline information, Option B is a strong second.
