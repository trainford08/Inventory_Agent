Ada layout — Option A vs B

Same review session (/teams/alpha/review/code-and-repos?ada=uses_lfs), two ways of fitting Ada into the existing dual-nav review layout.

Option A
3-column: Ada takes space
Ada renders as a third column inside the review layout. Main review column shrinks by ~300px when Ada is open, reflows when closed.
  • + Content never overlaps — everything stays visible at once.
  • + Ada feels like part of the review, not a modal.
  • − Main review column gets tight — item cards reflow.
  • − Fields the user just reviewed jump visually when Ada opens.
  • − On smaller screens this breaks down fast.
Feel: Ada is a peer column. Works best if Ada is expected to be open most of the time.

Option B · recommended
Slide-over: Ada floats on top
Ada is a fixed 300px panel anchored to the right edge that overlays the main column. Main layout is unchanged whether Ada is open or closed.
  • + Opening/closing Ada never reflows the review.
  • + Ada can cover the field it's discussing — direct spatial link.
  • + Simpler to ship; doesn't touch ReviewShell internals.
  • − Covers a slice of the review content while open.
  • − Feels more "modal" — closer to a chat drawer than a peer.
Feel: Ada is a helper you summon for a specific question, then dismiss. Matches the design's language ("helping with…").
