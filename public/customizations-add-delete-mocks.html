Customizations table — add/delete row UX

Three patterns for letting a champion add their team's own customizations and remove rows that don't apply.

Option A — Hover row actions + inline "Add" footer per category

Delete revealed on row hover. Each category has a dashed "+ Add" row at the bottom. Lowest-friction for one-off edits; reads as a regular table until you hover.

Recommendation: Option A. The customizations view is reference-heavy (champions read more than they edit), so revealing destructive actions on hover keeps the table calm; the per-category "+ Add" footer is low-friction and reinforces correct categorization. If multi-select / bulk deletion turns out to be common after testing, promote to Option B; otherwise Option A scales cleanly.
